# web 版 UI 規範（apps/website）

img-diff web 版の **見た目（ビジュアル／UI）の正本**。
ロジックは `packages/schema/SPEC.md`、実装アーキテクチャ（ワーカー・IndexedDB・権限）は
`apps/website/DESIGN.md`、**本書は配色・タイポ・余白・コンポーネントの規範**を定める。
迷ったら本書のトークン（§5）に従う。

姉妹プロダクト diffx（テキスト差分）と同じ設計言語＝**速い・簡潔・精確**を引き継ぐ。
ただし **主色は Teal**（diffx は靛藍）。理由は §4。

---

## 1. ユーザーと前提

- 日本ユーザー主体（中文など CJK もカバー）。重複・類似画像の整理が用途：
  - 写真フォルダの重複削除、素材ライブラリの近似画像検出、2 枚の見比べ。
- **プライバシー重視**（全クライアント計算・サーバなし）。
- **学習コストゼロ**：フォルダを選ぶ → 待つ → 結果を見る。
- 主に **日中の作業環境**（亮色既定）。暗色も同等に磨く。

## 2. ブランド人格

**速い・簡潔・精確。**

UI が伝える感情：高速で信頼でき、冷静で精確。派手にしない・媚びない。
**ツールそのものの仕事ぶりが最良の説得力**。

## 3. 設計原則（diffx から継承）

1. **速さがブランド** — 速く感じさせる。コンパクトな配置・明快な階層・無駄な遷移アニメなし。
2. **抑制が品質** — 全要素に存在理由を。装飾アイコン・冗長な説明文・「見栄えだけ」の物を置かない。
3. **精確が信頼** — 整列・間隔は一貫。**統計値（件数・容量・SSIM）は等幅数字**で揺れなく。
4. **静けさが専門性** — 注意を奪わない。ユーザーの**画像**が主役、UI は黒子。必要な時だけ現れる。
5. **亮色が日常** — 亮色既定が主体験。暗色は「色反転」では済ませず同等に磨く。

## 4. 審美方向

- **精緻に磨かれた抑制** — 粗野な「ただのツール」ではなく、細部まで詰めたミニマル。
- **亮色テーマ既定**、暗色は任意。
- **主色 = Teal（深青緑）**。冷静・清潔・「工具」寄り。
  画像 diff／SSIM ヒートマップは**赤・品紅**系で重ねるので、主色を Teal にして**赤系と補色で衝突させない**。
- **反面教師**：2024–2025 の「AI 生成ページ」風にしない（§9）。
- **アクセシビリティ：WCAG AAA** を目標（本文コントラスト 7:1 以上、§8）。

## 5. デザイントークン

実装は CSS カスタムプロパティで持つ。値は実装時にコントラストチェッカで最終検証する。

### 5.1 色 — 亮色（既定）

```
/* 背景・面 */
--bg:            #FBFDFD;   /* 青みのある冷白 */
--surface:       #FFFFFF;   /* カード・パネル */
--border:        #DCE6E5;   /* 罫線・区切り（冷灰） */
--border-strong: #C3D2D0;

/* テキスト（白背景で AAA） */
--text:          #0F2A27;   /* 主文（青み黒）   */
--text-secondary:#3D504D;   /* 副文（7:1 を満たす冷灰） */
--text-muted:    #5B6E6B;   /* 補助（大きめ文字・非必須情報のみ） */

/* 主色 Teal */
--primary:       #0F766E;   /* 塗り（ボタン背景・選択リング） */
--primary-hover: #0E6B63;
--primary-fg:    #FFFFFF;   /* 主色上の文字 */
--primary-text:  #115E59;   /* ★小さい teal 文字／リンクはこの濃さで（白背景 AAA 用） */

/* セマンティック */
--diff-overlay:  #FF2D55;   /* 画素 diff の重ね（品紅寄りの赤・teal と補色） */
--danger:        #B91C1C;   /* 削除など破壊的操作 */
--warning:       #92510A;   /* 注意（白背景 AAA 用に濃いめ） */
--focus-ring:    #0F766E;   /* フォーカス可視リング */
```

- **本文は `--text`／`--text-secondary`**（白背景で 7:1 以上＝AAA）。
- **Teal を「文字」に使うときは `--primary-text`**（`--primary` の塗り色は小文字だと 7:1 に届かない）。
  塗りボタンの白文字（`--primary-fg` on `--primary`）は大きめ太字 UI として扱う。

### 5.2 色 — 暗色

```
--bg:            #0B1413;
--surface:       #111E1C;
--border:        #1E2E2C;
--text:          #E7F1EF;
--text-secondary:#A9BDBA;
--primary:       #2DD4BF;   /* 暗色では明るい teal をアクセントに */
--primary-fg:    #06201D;   /* 明るい teal 上は暗い文字 */
--diff-overlay:  #FF5470;   /* 暗背景でも鮮明に */
```

切替は `prefers-color-scheme` 既定 + 手動トグル。**色反転で済ませない**（面・罫線・影を個別に設計）。

### 5.3 タイポグラフィ

```
--font-sans: ui-sans-serif, system-ui, "Segoe UI",
             "Hiragino Kaku Gothic ProN", "Noto Sans JP", Meiryo, sans-serif;
--font-mono: ui-monospace, "Cascadia Code", Consolas,
             "Noto Sans Mono CJK JP", monospace;
```

- **統計・件数・容量・SSIM・ハッシュは `--font-mono` + `font-variant-numeric: tabular-nums`**（原則 3：揺れない数字）。
- スケール（rem）：12 / 14（本文既定）/ 16 / 20 / 24 / 32。行間は本文 1.6、見出し 1.25。
- 太さは 400 / 500 / 600 の 3 段まで。極太・極細を多用しない。

### 5.4 余白・角丸・影・動き

```
--space: 4 / 8 / 12 / 16 / 24 / 32 / 48 (px, 4px グリッド)
--radius-sm: 6px   /* チップ・小要素 */
--radius:    8px   /* ボタン・入力・サムネ */
--radius-lg: 12px  /* カード・パネル */
--shadow-sm: 0 1px 2px rgba(15,42,39,.06);
--shadow-md: 0 2px 8px rgba(15,42,39,.08);
--ease: cubic-bezier(.2,0,0,1);
--dur:  120–160ms;
```

- **影は控えめ**（罫線 + 極薄影）。glow（発光）禁止。
- 動きは速く短く、`ease-out` 基調。**`prefers-reduced-motion` を尊重**して無効化可能に。
- 角丸は中庸（pill 乱用・直角どちらにも振らない）。

## 6. 画像特有の UI

- **サムネ網格**：正方形セル・`object-fit: cover`・`--border` 1px。
  **透過 PNG は市松（チェッカー）背景**で透明域を示す。選択は **`--primary` の 2px リング**（色だけに頼らずチェック印も付ける）。
- **重複グループ**：1 グループ = 1 カード。代表（残す 1 枚）を強調、他は選択削除候補。
  カードに **削減可能容量（reclaimableBytes）を等幅数字**で表示。
- **2 枚比較**：並べ表示 + スライダ重ね + **画素 diff モード**（`--diff-overlay` で差分を重ねる）。
  **SSIM 値は等幅数字**。diff の赤系は主色 teal と補色で読み分けやすい。
- **進捗**：線形バー + `3000 / 5000` の等幅表記。派手なスピナーは使わない。
- **削除導線**：破壊的操作は `--danger`。確認を挟む（perceptual 一致は自動削除しない＝SPEC §5）。

## 7. アクセシビリティ（WCAG AAA 目標）

- 本文テキストはコントラスト **7:1 以上**（§5 の `--text` 系）。teal 文字は `--primary-text`。
- **色を唯一の情報伝達にしない**：選択＝リング + チェック、diff＝色 + 記号/ラベル、状態＝色 + テキスト。
- **フォーカス可視**（`--focus-ring`）、全操作キーボード到達可能。
- フォーム・ボタンに適切な `aria` ラベル。サムネには意味のある `alt`（ファイル名等）。

## 8. アンチパターン（「AI 生成ページ」風を避ける）

やらない：

- グラデーション文字、発光カード、ネオン暗色テーマ。
- 紫〜ピンクの定番 AI グラデ、意味のない大きな角丸ピル乱用。
- 装飾だけの emoji・アイコン、意味のないホバー/スクロール演出。
- 中身のない「すごそうな」マイクロインタラクション。

代わりに：**冷静な teal × 冷灰、十分な余白、明快な階層、等幅の数字、最小限の影**。
画像そのものを主役にし、UI は静かに支える。
